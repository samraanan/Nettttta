# יומן שיחה ופיתוח - MyTrack V6
**תאריך:** 2 בפברואר 2026

## מטרות הסשן
1. **תיקון באג "עדכון חם" (Hot Update):** שיעורים שנוספו בעורך לא התעדכנו מיד בלוח המחוונים היומי, במיוחד עבור ימי ראשון שכבר עברו (לצורך ניקוד רטרואקטיבי).
2. **שינוי ממשק משתמש:** מעבר מתצוגת לשוניות (Tabs) לתצוגה שבועית מלאה (Grid) וביטול השמירה האוטומטית לטובת שמירה ידנית.
3. **פריסה וגיבוי:** העלאה ל-Surge וגיבוי הקוד ב-GitHub.

---

## מהלך העבודה והתיקונים

### 1. תיקון לוגיקת עדכון וניקוד
- **הבעיה:** בעת עריכת מערכת ביום שבת (למשל), המערכת עדכנה רק את יום ראשון *הבא*. המשתמש רצה לערוך את יום ראשון *שעבר* באותו שבוע כדי להזין ניקוד.
- **הפתרון:** שינינו את הלוגיקה ב-`storage.js` כך שהחישוב מתייחס לשבוע הנוכחי (ראשון עד שבת), גם אם היום כבר עבר.
- **תיקון טרנזקציה:** נתקלנו בשגיאת Firestore (`Read must come before Write`). סידרנו מחדש את סדר הפעולות בתוך הטרנזקציה ב-`updateTemplate` כדי לפתור זאת.

### 2. שכתוב עורך המערכת (`ScheduleEditor.jsx`)
- **ביטול שמירה אוטומטית:** המשתמש דיווח שהשמירה האוטומטית מבלבלת או לא אמינה.
- **תצוגת Grid:** במקום לעבור בין ימים, בנינו את הממשק מחדש להציג את כל 5 ימי הלימוד עמודה לצד עמודה.
- **כפתור שמירה:** הוספנו כפתור "שמור שינויים" בולט ששומר את כל השבוע בבת אחת בענן. הוספנו חיווי ויזואלי (ירוק/אפור) למצב השמירה.

### 3. אבטחה ופריסה
- **חסימת Google:** הדומיין הקודם סומן בטעות כמסוכן.
- **פעולה:** המערכת הועלתה לדומיין חדש ונקי: `https://my-track-v6-safe.surge.sh`.

### 4. גיבוי לקוד מקור (Git)
- הפרויקט אותחל כמאגר Git מקומי.
- בוצע חיבור ל-Remote ב-GitHub: `https://github.com/samraanan/Nettttta.git`.
- כל הקבצים הועלו לענף `main`.
